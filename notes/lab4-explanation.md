# Алгоритм розрахунку IP-адресації підмереж

## Теоретична основа

IP-адреса версії 4 (IPv4) являє собою 32-бітне число, що записується у вигляді чотирьох десяткових октетів, розділених крапками (наприклад, `10.0.0.0`). Маска підмережі — це також 32-бітне число, в якому старші біти (одиниці) визначають **мережеву частину** адреси, а молодші біти (нулі) — **хостову частину**.

```
IP-адреса:      10  .   0   .   0   .   0
У бінарному: 00001010.00000000.00000000.00000000

Маска /24:      255  .  255  .  255  .   0
У бінарному: 11111111.11111111.11111111.00000000
             |-------- мережа ---------|--хости-|
```

---

## Крок 1 — Запис маски у бінарному вигляді

Маска підмережі записується у префіксній нотації `/n`, де `n` — кількість одиничних бітів. Решта `32 - n` бітів є нульовими і визначають хостову частину.

```
/24 → 11111111.11111111.11111111.00000000 → 255.255.255.0
/29 → 11111111.11111111.11111111.11111000 → 255.255.255.248
/28 → 11111111.11111111.11111111.11110000 → 255.255.255.240
```

---

## Крок 2 — Визначення кількості хостів

Кількість доступних хостових адрес у підмережі визначається за формулою:

```
N = 2ⁿ - 2
```

де `n` — кількість нульових бітів у масці (хостова частина). Від загальної кількості адрес (`2ⁿ`) віднімається 2, оскільки:

- перша адреса діапазону є **адресою мережі** і не призначається хостам
- остання адреса діапазону є **широкомовною адресою (broadcast)** і також не призначається хостам

Таблиця вибору маски залежно від кількості необхідних хостів:

| Маска           | Нотація | Бітів під хости (n) | Кількість хостів (2ⁿ - 2) |
| --------------- | ------- | ------------------- | ------------------------- |
| 255.255.255.252 | /30     | 2                   | 2                         |
| 255.255.255.248 | /29     | 3                   | 6                         |
| 255.255.255.240 | /28     | 4                   | 14                        |
| 255.255.255.224 | /27     | 5                   | 30                        |
| 255.255.255.192 | /26     | 6                   | 62                        |
| 255.255.255.128 | /25     | 7                   | 126                       |
| 255.255.255.0   | /24     | 8                   | 254                       |

---

## Крок 3 — Визначення кроку між підмережами

Крок між підмережами визначається за **останнім ненульовим октетом маски** за формулою:

```
Крок = 256 - значення останнього ненульового октету маски
```

Крок застосовується саме до того октету адреси, який відповідає останньому ненульовому октету маски.

Приклади:

```
/29 → маска 255.255.255.248 → останній ненульовий октет = 248 (4-й)
      крок = 256 - 248 = 8
      підмережі: 10.0.0.0, 10.0.0.8, 10.0.0.16 ...

/28 → маска 255.255.255.240 → останній ненульовий октет = 240 (4-й)
      крок = 256 - 240 = 16
      підмережі: 10.0.0.0, 10.0.0.16, 10.0.0.32 ...

/20 → маска 255.255.240.0   → останній ненульовий октет = 240 (3-й!)
      крок = 256 - 240 = 16, застосовується до 3-го октету
      підмережі: 10.0.0.0, 10.0.16.0, 10.0.32.0 ...
```

---

## Крок 4 — Визначення адреси мережі

Адреса мережі визначається операцією побітового **AND** між IP-адресою та маскою підмережі. У цій операції біт результату дорівнює 1 лише тоді, коли обидва відповідні біти операндів дорівнюють 1.

```
Таблиця істинності AND:
0 AND 0 = 0
0 AND 1 = 0
1 AND 0 = 0
1 AND 1 = 1
```

Приклад для `10.0.0.5` з маскою `/29`:

```
10.0.0.5        → 00001010.00000000.00000000.00000101
255.255.255.248 → 11111111.11111111.11111111.11111000
AND результат   → 00001010.00000000.00000000.00000000
               =  10.0.0.0  ← адреса мережі
```

---

## Крок 5 — Визначення широкомовної адреси (broadcast)

Широкомовна адреса визначається заміною всіх хостових бітів (нулів маски) на одиниці в адресі мережі, або за формулою:

```
Broadcast = адреса мережі + крок - 1
```

Приклад для `10.0.0.0/29` (крок = 8):

```
Broadcast = 10.0.0.0 + 8 - 1 = 10.0.0.7
```

Перевірка у бінарному вигляді:

```
Адреса мережі:  00001010.00000000.00000000.00000000
Хостові біти → 00001010.00000000.00000000.00000111
Broadcast    =  10.0.0.7  ✅
```

---

## Крок 6 — Визначення діапазону хостових адрес

```
Перший хост   = адреса мережі + 1
Останній хост = broadcast - 1
```

Приклад для `10.0.0.0/29`:

```
Адреса мережі: 10.0.0.0  (не призначається хостам)
Перший хост:   10.0.0.1
Останній хост: 10.0.0.6
Broadcast:     10.0.0.7  (не призначається хостам)
```

---

## Крок 7 — Визначення наступної підмережі

```
Наступна мережа = поточна адреса мережі + крок
```

---

## Зведена таблиця алгоритму

| Крок | Дія                                | Формула                                  |
| ---- | ---------------------------------- | ---------------------------------------- |
| 1    | Записати маску в бінарному вигляді | `/n → n` одиниць, решта нулі             |
| 2    | Визначити кількість хостів         | `2ⁿ - 2`, де `n` — кількість нулів маски |
| 3    | Визначити крок між підмережами     | `256 - останній ненульовий октет маски`  |
| 4    | Визначити адресу мережі            | `IP AND маска` (побітово)                |
| 5    | Визначити broadcast                | `адреса мережі + крок - 1`               |
| 6    | Визначити діапазон хостів          | від `мережа + 1` до `broadcast - 1`      |
| 7    | Визначити наступну підмережу       | `поточна мережа + крок`                  |

---

## Застосування алгоритму: Варіант А

**Умова:** розмістити **8 ПК** в одній підмережі. Вихідна мережа: `10.0.0.0`.

### Крок 1 — Вибір маски

Необхідно знайти маску, за якої `2ⁿ - 2 ≥ 8`:

```
n = 3 → 2³ - 2 = 6   ❌ замало
n = 4 → 2⁴ - 2 = 14  ✅ достатньо → /28
n = 8 → 2⁸ - 2 = 254 ✅ з запасом → /24
```

Обираємо `/24` (`255.255.255.0`) — найпоширеніша маска для локальних мереж,
забезпечує 254 хости та є стандартом для навчальних і малих офісних мереж.

### Крок 2 — Крок між підмережами

```
256 - 0 = 256
```

Для `/24` весь четвертий октет є хостовим, тому в межах мережі `10.0.0.0/24`
існує лише одна підмережа.

### Крок 3 — Адреса мережі (AND)

```
10.0.0.0      → 00001010.00000000.00000000.00000000
255.255.255.0 → 11111111.11111111.11111111.00000000
AND           → 00001010.00000000.00000000.00000000
             =  10.0.0.0  ← адреса мережі
```

### Крок 4 — Broadcast

```
Broadcast = 10.0.0.0 + 256 - 1 = 10.0.0.255
```

### Крок 5 — Діапазон хостів

```
Перший хост:   10.0.0.1
Останній хост: 10.0.0.254
```

### Результат — Варіант А

| ---- | --------- | ------------- | ------------- | ---------- |
| ПК 1 | 10.0.0.1  | 255.255.255.0 | 10.0.0.0      | 10.0.0.255 |
| ПК 2 | 10.0.0.2  | 255.255.255.0 | 10.0.0.0      | 10.0.0.255 |
| ПК 3 | 10.0.0.3  | 255.255.255.0 | 10.0.0.0      | 10.0.0.255 |
| ПК 4 | 10.0.0.4  | 255.255.255.0 | 10.0.0.0      | 10.0.0.255 |
| ПК 5 | 10.0.0.5  | 255.255.255.0 | 10.0.0.0      | 10.0.0.255 |
| ПК 6 | 10.0.0.6  | 255.255.255.0 | 10.0.0.0      | 10.0.0.255 |
| ПК 7 | 10.0.0.7  | 255.255.255.0 | 10.0.0.0      | 10.0.0.255 |
| ПК 8 | 10.0.0.8  | 255.255.255.0 | 10.0.0.0      | 10.0.0.255 |

Всі ПК знаходяться в одній мережі `10.0.0.0/24` → вільно спілкуються між собою. ✅

## Застосування алгоритму: Варіант Б

**Умова:** розбити **8 ПК** на дві ізольовані підмережі по **4 ПК**. Вихідна мережа: `10.0.0.0`.

### Крок 1 — Вибір маски

Необхідно знайти маску, за якої `2ⁿ - 2 ≥ 4`:

```
n = 2 → 2² - 2 = 2  ❌ замало
n = 3 → 2³ - 2 = 6  ✅ достатньо → /29 (255.255.255.248)
```

Обираємо `/29` — мінімальна маска, що вміщує 4 хости (з двома резервними місцями).

### Крок 2 — Крок між підмережами

```
256 - 248 = 8
```

Підмережі починаються з адрес четвертого октету, кратних 8: `0`, `8`, `16`, `24` тощо.

---

### Підмережа А — ПК 1–4

**Адреса мережі (AND):**

```
10.0.0.0        → 00001010.00000000.00000000.00000000
255.255.255.248 → 11111111.11111111.11111111.11111000
AND             → 00001010.00000000.00000000.00000000
               =  10.0.0.0  ← адреса мережі А
```

**Broadcast:**

```
10.0.0.0 + 8 - 1 = 10.0.0.7
```

**Діапазон хостів:**

```
Перший хост:   10.0.0.1
Останній хост: 10.0.0.6
```

---

### Підмережа Б — ПК 5–8

**Адреса мережі** (застосовуємо крок 7):

```
10.0.0.0 + 8 = 10.0.0.8  ← адреса мережі Б
```

Перевірка через AND:

```
10.0.0.8        → 00001010.00000000.00000000.00001000
255.255.255.248 → 11111111.11111111.11111111.11111000
AND             → 00001010.00000000.00000000.00001000
               =  10.0.0.8  ✅
```

**Broadcast:**

```
10.0.0.8 + 8 - 1 = 10.0.0.15
```

**Діапазон хостів:**

```
Перший хост:   10.0.0.9
Останній хост: 10.0.0.14
```

---
