# Підказки до лабораторної роботи №10

## Завдання 1. Побудова топології в Packet Tracer

### Крок 1. Додайте обладнання

**Маршрутизатори:**

- Категорія **Routers** → модель **2811** → додайте на робочу область, назвіть **R0**
- Категорія **Routers** → модель **1841** → додайте на робочу область, назвіть **R1**

**Комутатор:**

- Категорія **Switches** → модель **2960-24TT** → додайте на робочу область, назвіть **SW0**

**Кінцеві пристрої:**

- Категорія **End Devices** → **PC-PT** → додайте два комп'ютери: **PC0** і **PC1**
- Категорія **End Devices** → **Server-PT** → додайте сервер, назвіть **DNS-WEB**

---

### Крок 2. Додайте модулі для серійних інтерфейсів

Обидва маршрутизатори не мають серійних інтерфейсів за замовчуванням — потрібно додати модуль **WIC-2T**.

**Для R0 (2811):**

1. Клікніть на R0 → вкладка **Physical**
2. Вимкніть живлення (кнопка збоку)
3. Знайдіть модуль **WIC-2T** у списку зліва та перетягніть його у слот **WIC3** (крайній правий/верхній слот, щоб отримати інтерфейс Se0/3/0)
4. Увімкніть живлення

**Для R1 (1841):**

1. Клікніть на R1 → вкладка **Physical**
2. Вимкніть живлення
3. Перетягніть модуль **WIC-2T** у вільний слот (отримаєте Se0/1/0)
4. Увімкніть живлення

> ⚠️ Якщо після додавання модуля серійний інтерфейс має іншу назву (наприклад Se0/0/0 замість Se0/3/0) — це залежить від того, в який слот встановлено модуль. Перевірте назву інтерфейсу командою `show ip interface brief` і використовуйте саме ту назву, що відображається в Packet Tracer.

---

### Крок 3. З'єднайте пристрої кабелями

Використовуйте інструмент **Connections** (блискавка в нижній панелі).

| З'єднання    | Тип кабелю                       | Інтерфейси                    |
| ------------ | -------------------------------- | ----------------------------- |
| PC0 ↔ SW0    | Copper Straight-Through (прямий) | PC0 Fa0 → SW0 будь-який порт  |
| PC1 ↔ SW0    | Copper Straight-Through (прямий) | PC1 Fa0 → SW0 будь-який порт  |
| SW0 ↔ R0     | Copper Straight-Through (прямий) | SW0 будь-який порт → R0 Fa0/0 |
| R0 ↔ R1      | Serial DCE/DTE                   | R0 Se0/3/0 → R1 Se0/1/0       |
| R1 ↔ DNS-WEB | Copper Straight-Through (прямий) | R1 Fa0/0 → DNS-WEB Fa0        |

> 💡 **Серійний кабель:** у Packet Tracer оберіть **Serial DCE** — кінець із позначкою «clock» (DCE) підключайте до **R0**. Це означає, що саме R0 задаватиме тактову частоту командою `clock rate`.
>
> Перевірити який кінець є DCE можна командою `show controllers Se0/3/0` на R0 — у виводі буде вказано `DCE`.

## Завдання 2. Налаштування IP-адрес

### На маршрутизаторах (R0, R1)

**Загальна послідовність команд:**

```
Router> enable
Router# configure terminal
Router(config)# hostname R0

R0(config)# interface fa0/0
R0(config-if)# ip address <IP-адреса> <маска>
R0(config-if)# no shutdown
R0(config-if)# exit

R0(config)# interface se0/3/0
R0(config-if)# ip address <IP-адреса> <маска>
R0(config-if)# clock rate 64000
R0(config-if)# no shutdown
R0(config-if)# exit
```

**Конкретні команди для R0:**

```
R0(config)# interface fa0/0
R0(config-if)# ip address 192.168.10.1 255.255.255.0
R0(config-if)# no shutdown
R0(config-if)# exit

R0(config)# interface se0/3/0
R0(config-if)# ip address 117.0.0.1 255.255.255.252
R0(config-if)# clock rate 64000
R0(config-if)# no shutdown
R0(config-if)# exit
```

**Конкретні команди для R1:**

```
R1(config)# interface fa0/0
R1(config-if)# ip address 219.0.0.1 255.255.255.252
R1(config-if)# no shutdown
R1(config-if)# exit

R1(config)# interface se0/1/0
R1(config-if)# ip address 117.0.0.2 255.255.255.252
R1(config-if)# no shutdown
R1(config-if)# exit
```

> ⚠️ **Команда `clock rate 64000`** вводиться лише на **R0** — на DCE-кінці серійного з'єднання. На R1 (DTE-кінець) вводити її не потрібно.

---

### На комп'ютерах (PC0, PC1)

PC0 і PC1 отримують мережеві налаштування **автоматично по DHCP** (буде налаштовано пізніше на R0). На цьому етапі додаткових дій не потрібно.

Щоб увімкнути DHCP на комп'ютері після налаштування сервера:

- Клікніть на PC → вкладка **Desktop** → **IP Configuration** → оберіть **DHCP**

---

### На сервері DNS-WEB

Клікніть на сервер → вкладка **Desktop** → **IP Configuration** → оберіть **Static** і введіть дані вручну.

| Поле            | Значення        |
| --------------- | --------------- |
| IP Address      | 219.0.0.2       |
| Subnet Mask     | 255.255.255.252 |
| Default Gateway | 219.0.0.1       |

---

### Перевірка після налаштування

Виконайте на кожному маршрутизаторі:

```
R0# show ip interface brief
```

У стовпцях **Status** і **Protocol** мають бути значення `up` / `up` для всіх налаштованих інтерфейсів.  
Якщо бачите `administratively down` — інтерфейс не активовано командою `no shutdown`.
